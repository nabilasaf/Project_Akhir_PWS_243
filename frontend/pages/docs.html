Welcome to the GameVault API! This API provides access to comprehensive game data including titles, genres, platforms, ratings, and more. Follow this guide to get started.

üîê Authentication

All API requests require authentication using your API key. Include your key in the X-API-Key header of every request.

// Example request with authentication fetch('https://api.gamevault.com/v1/games', { headers: { 'X-API-Key': 'your_api_key_here' } }) .then(response => response.json()) .then(data => console.log(data));

‚ö†Ô∏è Keep Your API Key Secret

Never expose your API key in client-side code or public repositories. Use environment variables or secure backend storage.

üì° API Endpoints

Get All Games

GET /api/v1/games

Retrieve a list of all available games with optional search filtering.

Query Parameters

Parameter Type Description
search string Search games by title or genre
limit integer Number of results per page (default: 50)

Example Response

[ { "id": 1, "title": "Elden Ring", "genre": "RPG", "platform": "PC, PlayStation, Xbox", "rating": 4.9, "apiAvailable": true } ]

Get Game by ID

GET /api/v1/games/:id

Retrieve detailed information about a specific game.

Example Response

{ "id": 1, "title": "Elden Ring", "genre": "RPG", "platform": "PC, PlayStation, Xbox", "rating": 4.9, "description": "Epic fantasy action RPG", "apiAvailable": true }

‚ùå Error Handling

The API uses standard HTTP status codes to indicate success or failure:

Status Code Meaning
200 Success - Request completed successfully
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing API key
429 Too Many Requests - Rate limit exceeded
500 Server Error - Something went wrong on our end
